Data Communication

Data communication is the epitome of how data is transferred from sending messages to presenting websites. In this section, we will be looking at how data is packaged and sent, examining different protocols that govern the very essence of data transmission across the vast platform we call the internet. We will go over the different protocols and walk through some code that orchestrates this delivery.

UDP protocols

What is it?

UDP stands for User Datagram Protocol and is ultimately a connectionless protocol that sends out a message to an endpoint, which could be another device. The required parameters we need in order to send data are an IP address and an unused port number. An IP address is just a 32-bit virtual address where your device resides. The Port Number is just an access point that allows your device to access. Think of it like a channel on a TV. If the Port Number is different, then the packet will be dropped and lost forever.

Coding time...

                udpClnt = new UdpClient();
                byte[] sendBytes = Encoding.ASCII.GetBytes(message);
                udpClnt.Client.Bind(new IPEndPoint(IPAddress.Any, sendPortMessage));
                udpClnt.Send(sendBytes, sendBytes.Length, ipAddress, sendPortMessage);

Above is an example of how to initialize a UDP socket, which in C# is very easy and can be done with the System.Net library. As UDP needs to know its destination, such as an IP address and a port number, it will have to bind this information to it and send it right away. There is no need for a three-way handshake or confirmation with the end device. Therefore, reliability is out the window. As it is connectionless, it will not detect the other device before sending. This form of transportation or protocol is not optimal for crucial data like images, user credentials, and so on.

Compared to its counterpart, this connectionless protocol only requires the bare bones of information. The IP address of the intended destiation and its own, length of the data or payload, and the error detection algorithm which in this case is checksum.
